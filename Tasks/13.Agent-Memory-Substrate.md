## Task 13: memory substrate deployment

Mission:
Deploy and configure per-agent MemPalace instances for OpenClaw and Hermes, and a shared Honcho secondary memory substrate for cross-agent continuity. NoesisPraxis is the Hermes supervisor agent for the Noesis Universe infrastructure stack, and memory must be split so each agent has isolated primary memory while sharing a unified secondary substrate.

Canonical references:
- MemPalace: https://github.com/mempalace/mempalace
- MemPalace MCP integration: https://mempalaceofficial.com/guide/mcp-integration.html
- MemPalace OpenClaw integration: https://mempalaceofficial.com/guide/openclaw.html
- MemPalace hooks: https://mempalaceofficial.com/guide/hooks.html
- Honcho: https://github.com/plastic-labs/honcho

- Honcho SDK: https://honcho.dev/docs/v3/documentation/reference/sdk
- Honcho CLI: https://honcho.dev/docs/v3/documentation/reference/cli

Requirements:
1. Give each agent its own dedicated MemPalace instance and configuration.
2. Deploy Honcho as a shared secondary memory substrate for unified context and cross-agent continuity.
3. Keep MemPalace per-agent memory isolated by default.
4. Make Honcho clearly shared and centrally managed.
5. Add Ansible playbooks, roles, templates, or validation tasks to deploy and validate the memory stack.

Acceptance criteria:
- OpenClaw has its own MemPalace instance.
- Hermes has its own MemPalace instance.
- Honcho is deployed as a shared secondary memory substrate.
- Validation confirms the memory topology is correct and reachable.
