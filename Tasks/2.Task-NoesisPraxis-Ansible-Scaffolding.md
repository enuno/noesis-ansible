Task:
Generate the two top-level orchestration playbooks `playbooks/site.yml` and `playbooks/master-stack.yml` for the NoesisPraxis Ansible monorepo. These playbooks must wire together the entire repository skeleton and provide the phase-by-phase orchestration path for the full multi-agent stack.

The playbooks must not implement the full business logic of each subsystem. Instead, they must include and sequence the appropriate roles, task files, and validation steps so the stack can be deployed in a controlled, modular, and reversible manner.

Repository context:
The repo already contains the following top-level areas and roles:
- ACP registry
- ANS
- Agent Registry dashboard

- MCPJungle
- Clawvisor
- Telegram
- OpenClaw
- Hermes
- macOS ClawDev
- macOS HermesDev
- Tailscale
- Backup
- Validation

These playbooks are the primary orchestration entry points for the repo.

Design requirements:
- `site.yml` must be the user-facing entry point for normal execution.
- `master-stack.yml` must be the phase-oriented master orchestrator that can run the stack in modular stages.
- Both playbooks must support tags so operators can run individual phases.
- Both playbooks must support local, tailscale, macOS, and generic dockerized paths through inventory and variable selection.
- Both playbooks must be idempotent and safe to rerun.
- Both playbooks must include validation hooks after each major phase.
- Both playbooks must make rollback and maintenance paths explicit.
- Both playbooks must use clean `import_playbook` or `include_role` patterns without hardcoding implementation details.
- Both playbooks must preserve ordering dependencies:
  - registry and identity foundation first
  - tool access and security second
  - runtime deployments third
  - macOS-specific services and Tailscale fourth
  - communications and sync fifth
  - validation and backup last

Required `site.yml` behavior:
- Serve as the default operator entry point.
- Select inventory and environment through variables, not hardcoded assumptions.
- Run the master stack in a sane default order.
- Allow tags like `foundation`, `security`, `runtime`, `macos`, `tailscale`, `telegram`, `validate`, `backup`, and `rollback`.
