Use Ansible as the primary orchestration tool for all Noesis back-end infrastructure and maintenance. The repository at ~/projects/noesis-ansible/ is the source of truth for deployment, configuration, upgrades, backups, registry synchronization, and maintenance of the Noesis multi-agent environment, including MCP servers and related services directly consumed by AI agents. All secrets must originate in Bitwarden Secrets Manager and be fetched locally with bws, then imported into Ansible Vault so routine Ansible execution does not need to query Bitwarden externally. All infrastructure changes must be expressed as composable, idempotent Ansible playbooks and roles.
