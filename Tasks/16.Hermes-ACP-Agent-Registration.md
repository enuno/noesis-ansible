Mission:
Configure and register Hermes agents for ACP operation, using the Hermes ACP skill and official Hermes documentation as the implementation guide. The goal is to provision Hermes agents so they can be driven externally over ACP, exposed safely as controllable agents, and registered into the appropriate discovery and registry layers for cross-agent coordination.

Primary references:
- Hermes ACP skill / implementation guide:
  https://github.com/Rainhoole/hermes-agent-acp-skill
- ACP internals:
  https://hermes-agent.nousresearch.com/docs/developer-guide/acp-internals
- ACP user feature docs:
  https://hermes-agent.nousresearch.com/docs/user-guide/features/acp

Objectives:
- Generate the configuration and registration workflow needed to enable Hermes agents for ACP.
- Ensure each Hermes agent can start, serve, and be addressed over ACP stdio JSON-RPC.
- Ensure agents are registered with the platform’s discovery and registry systems where applicable.
- Keep the configuration reusable across generic Hermes agents and specialized Noesis* variants.
- Make the setup production-safe, auditable, and reversible.

Scope:
- ACP adapter and transport configuration.
- Hermes agent startup wiring for ACP.
- Agent identity metadata and registry entries.
- Gateway or bridge exposure for external control.
- Multi-agent discovery and supervisor routing.
- Documentation and operational runbooks for registration, validation, and teardown.

Hermes ACP behavior requirements:
- Treat `hermes acp` as the primary ACP entrypoint.
- Configure the ACP adapter to run as an async JSON-RPC stdio server.
- Preserve session lifecycle, tool-call events, approvals, and streaming event handling.
- Validate startup paths such as version checks, setup checks, environment loading, and transport initialization.
- Ensure the agent can participate in external orchestration without leaking privileged internal state.

Configuration requirements:
- Create reusable configuration modules or playbook roles for:
  - ACP enablement,
  - Hermes agent identity,
  - adapter startup,
  - transport binding,
  - environment preparation,
  - registry enrollment,
  - health checks and lifecycle supervision.
- Parameterize:
  - agent canonical name,
  - agent role,
  - ACP command,
  - session naming,
  - environment file location,
  - supervisor binding,
  - registry endpoints,
  - auth or credential references.
- Keep secrets out of plain configuration.
- Use vaulting or external secret management for protected values.

Registration requirements:
- Register the Hermes agent with the relevant agent registry / discovery layer used by the stack.
- Publish metadata that includes:
  - canonical identity,
  - ACP support,
  - capabilities,
  - transport type,
  - supervisor relationship,
  - supported interaction modes,
  - environment scope.
- If the environment has A2A or ACP-compatible discovery, ensure Hermes agents are discoverable there as well.
- Support lifecycle updates so registry entries are refreshed on deploy, upgrade, or identity changes.

Operational requirements:
- Validate the ACP adapter before marking the agent ready.
- Verify session creation, prompt submission, streaming output, and tool-call handling.
- Confirm that approvals and permission flows behave correctly.
- Provide rollback and deregistration steps.
- Log registration actions, startup checks, and adapter failures.

Security and boundaries:
- Do not expose internal secrets, tokens, or private keys through ACP.
- Use least privilege for agent execution and registry writes.
- Require explicit approval for sensitive tool calls or destructive actions.
- Treat registration and control-plane metadata as operationally sensitive.
- Preserve Hermes’ built-in safety and approval model during ACP exposure.

Deliverables:
1) Configurations or playbook fragments to enable ACP on Hermes agents.
2) Registry enrollment logic and metadata templates.
3) Validation checks for ACP readiness and session behavior.
4) Documentation for deploy, update, rollback, and deregistration.
5) A reusable pattern for generic Hermes agents and specialized Noesis* agents.

Acceptance criteria:
- Hermes agents can be started with ACP enabled.
- External clients can connect over ACP stdio JSON-RPC.
- Hermes agents are discoverable through the intended registry layer.
- NoesisPraxis can supervise and route work to the registered agents.
- The setup is idempotent, secure, and repeatable.
- The implementation follows the Hermes ACP adapter lifecycle and user-facing ACP behavior described in the docs.

Execution policy:
- Read the ACP skill and docs first.
- Prefer documented startup and adapter paths over ad hoc wiring.
- Fail fast on startup or registration errors.
- Keep the adapter thin and the configuration explicit.
- Reuse common templates across agent variants.
